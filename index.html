Specifications

I represent Google in the W3C's Web Application Security working group, and I'm responsible for the design and implementation of a number of specifications (listed here in approximate order of maturity/implementation):

  • Content Security Policy Level 2 (with Adam Barth and Daniel Veditz)
  • Mixed Content
  • Secure Contexts (with Yan Zhu)
  • Upgrade Insecure Requests
  • Credential Management Level 1
  • Content Security Policy Level 3
  • Clear Site Data
  • Content Security Policy: Embedded Enforcement
  • Signature-based Resource Loading Restrictions
  • Reporting API
  • CORS and RFC1918
  • Origin Policy

These documents almost always involve deeper changes to the platform, reflected in my contributions to WHATWG’s HTML and Fetch standards.

I also do a bit of related work in the IETF's HTTP working group, mostly related to cookies:

  • Initial Assignment for the Content Security Policy Directives Registry (RFC 7762)
  • RFC6265bis refreshes the cookies RFC by incorporating the following documents I'm responsible for:
    • Deprecate modification of `secure` cookies from non-secure origins
    • Cookie Prefixes
    • Same-Site Cookies (with Mark Goodwin)
  • Cookie Priority (with Eric Wright and Samuel Huang)
  • Let `localhost` be localhost
